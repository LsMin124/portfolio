02b

열-냉각 PID 제어 — 폐루프를 처음부터 끝까지

코딩+제어 ~6h v2 §4.2(e) 직결 — 반나절 실습 = 모듈 초안

학습 목표

  • 피드백(폐루프) 제어의 구조와 P·I·D 각 항의 역할을 설명할 수 있다.
  • 1차 열 모델을 이산 시간으로 시뮬레이션하고, 직접 구현한 PID로 목표 온도를 추종시킨다.
  • anti-windup이 왜 필요한지 현상으로 보고 구현한다.
  • 수동 튜닝 절차(P → I → D)로 게인을 잡고 정착시간·오버슈트를 측정한다.
  • 열화 시나리오에서 SPC ±3σ 경보를 만들고 precision/recall로 채점한다.
프로젝트와의 연결 — v2 §4.2(e), 면접 방어 6·7·8번

v2 개정으로 열-냉각 PID 모듈이 코어에 들어왔다. 이 스텝의 실습 코드가 그 모듈의 초안이며, "기계공학이 어디 쓰였나"의 첫 번째 답(모델링+제어)이 된다. 핵심 통찰 하나를 미리 심어두면 — 제어가 좋을수록 고장은 늦게 보인다. 그래서 온도가 아니라 제어 노력을 감시한다. 이 문장을 코드로 증명하는 것이 이 스텝의 종착지다.

1. 폐루프의 해부 — 다섯 단어

목표(setpoint)와 측정(measurement)의 차이가 오차(error)이고, 제어기가 오차를 보고 조작량(u)을 계산해 플랜트(plant)에 가하면, 플랜트의 반응이 다시 측정으로 돌아온다. 개루프(정해진 출력을 틀어놓는 것)와의 차이는 단 하나 — 결과를 보고 스스로 고친다는 것.

  • P (비례) — 지금 오차에 비례해 민다. 크면 빠르지만 진동하고, 혼자서는 정상상태 오차가 남는다.
  • I (적분) — 오차의 누적을 민다. 정상상태 오차를 지워주지만, 과하면 출렁이고 포화와 만나면 windup 사고를 낸다.
  • D (미분) — 오차의 변화 속도에 브레이크를 건다. 진동을 눌러주지만 노이즈에 민감하다.

2. 플랜트 — 1차 열 모델

설비 온도의 최소 물리: 부하에 비례해 열이 들어오고, 냉각계가 온도차에 비례해 열을 내보낸다. 열용량 C로 나눠 적분하면 온도 궤적이 나온다 (오일러 적분, 시간 스텝 1분):

plant.py — 숫자는 전부 시작점, 튜닝이 과제다
def step_plant(T, load, u, k_cool, dt=1.0, T_amb=25.0, C=40.0):
    """1차 열 모델 한 스텝.
    T: 현재 온도, load: 부하(0~1), u: 냉각 출력(0~1),
    k_cool: 방열 효율 — 열화 시 이 값이 떨어진다."""
    q_in  = 8.0 * load                    # 부하 비례 발열
    q_out = k_cool * u * (T - T_amb)      # 냉각계 방출
    return T + (q_in - q_out) * dt / C

이 모델의 정직한 성질: u를 고정하면(개루프) 부하가 바뀔 때마다 온도가 표류한다. 미션 1번에서 이걸 직접 확인하고 시작하라 — 폐루프가 왜 필요한지 몸으로 알게 된다.

3. 제어기 — 이산 PID와 anti-windup

pid.py — 직동형 (과열이면 냉각을 올린다)
class PID:
    def __init__(self, kp, ki, kd, u_min=0.0, u_max=1.0):
        self.kp, self.ki, self.kd = kp, ki, kd
        self.u_min, self.u_max = u_min, u_max
        self.integral = 0.0
        self.prev_meas = None

    def update(self, setpoint, meas, dt):
        e = meas - setpoint          # 양수 = 과열 → 냉각 증가
        d = 0.0
        if self.prev_meas is not None:
            d = (meas - self.prev_meas) / dt   # 측정값 미분(킥 방지)
        self.prev_meas = meas

        u_raw = self.kp * e + self.ki * self.integral + self.kd * d
        u = min(max(u_raw, self.u_min), self.u_max)

        if u == u_raw:               # 포화가 아닐 때만 적분 (anti-windup)
            self.integral += e * dt
        return u

두 가지 실전 디테일이 이미 들어 있다:

  • 측정값 미분 — 오차 대신 측정값을 미분하면 목표값을 바꿀 때 D항이 튀는 "미분 킥"이 사라진다.
  • 조건부 적분(anti-windup) — 출력이 포화(0 또는 1에 붙음)된 동안에도 적분이 계속 쌓이면, 상황이 풀린 뒤 한참 동안 엉뚱한 출력을 낸다. 포화 중 적분을 멈추는 게 최소 처방. 미션에서 이걸 끄고 켜서 차이를 직접 봐라.

4. 수동 튜닝 — 절차를 문서화하는 것까지가 튜닝

  1. P만 (Ki=Kd=0): Kp를 올려가며 응답을 본다. 빨라지다가 진동 직전이 되는 지점을 기록.
  2. I 추가: 남아 있는 정상상태 오차가 사라질 만큼만 Ki를 올린다. 출렁이면 과했다.
  3. D는 선택: 오버슈트가 거슬릴 때만 소량. 이 시스템(1차+느림)에선 0이어도 충분할 수 있다 — "왜 D를 안 썼나"도 답이 되는 설계 결정이다.
  4. 각 단계에서 정착시간, 최대 오버슈트, 정상상태 오차를 기록한다 — v2 §8의 실측 지표이고, 면접 방어 7번의 답이다.

정석 기법이 궁금하면 Ziegler–Nichols를 검색해서 비교표 한 줄로 언급하라 — "수동 튜닝과 ZN 초기값을 비교했다"는 깊이의 증거가 된다.

5. 열화, 그리고 제어가 고장을 가리는 현상

여기가 이 스텝의 하이라이트다. 방열 효율 k_cool을 서서히 낮추면:

degradation_run.py — 8시간 조업 + 4시간 뒤 열화 시작
T, k_cool = 25.0, 1.0
pid = PID(kp=0.08, ki=0.02, kd=0.0)      # 시작점 — 직접 튜닝할 것
log = []

for minute in range(480):
    load = 0.8
    if minute >= 240:                     # 열화 시작
        k_cool = max(0.12, k_cool - 0.004)
    u = pid.update(setpoint=60.0, meas=T, dt=1.0)
    T = step_plant(T, load, u, k_cool)
    log.append((minute, T, u, k_cool))

그래프를 그려 보면 3막 구조가 보인다:

  1. 방어기 — 열화가 시작돼도 온도는 멀쩡하다. 대신 냉각 출력 u가 슬금슬금 올라간다. 제어 노력 상승 추세가 유일한 조기 징후.
  2. 포화 — u가 1.0에 붙는다. 제어기가 할 수 있는 게 없어진다.
  3. 이탈 — 그제서야 온도가 목표를 벗어난다. 온도 경보는 후행 지표였던 것.

6. 경보와 채점 — SPC ±3σ, 그리고 오탐

임계값은 감으로 정하지 않는다. 정상 운전 구간(warm-up 이후 ~ 열화 전)의 평균과 표준편차로 관리한계를 세운다:

alarm.py — 온도·제어 노력 각각에 관리한계
import numpy as np

normal = [(T, u) for t, T, u, k in log if 60 <= t < 240]
T_arr = np.array([x[0] for x in normal])
u_arr = np.array([x[1] for x in normal])

ucl_T = T_arr.mean() + 3 * T_arr.std(ddof=1)   # 온도 상한 관리한계
ucl_u = u_arr.mean() + 3 * u_arr.std(ddof=1)   # 제어 노력 관리한계

alarms_T = [t for t, T, u, k in log if T > ucl_T]
alarms_u = [t for t, T, u, k in log if u > ucl_u]
# 관측: alarms_u가 alarms_T보다 훨씬 먼저 울린다 (조기 vs 후행)

채점은 반드시 쌍으로: 열화 구간(t ≥ 240)의 경보는 TP, 정상 구간의 경보는 FP로 놓고 precision / recall / 오경보 횟수 / 탐지 리드타임을 함께 보고한다. 리드타임만 보면 임계값을 낮춰 일찍 울리게 하는 치팅이 가능하고, 그 순간 FP가 폭증한다 — 이 트레이드오프를 표로 보여주는 것 자체가 면접 방어다 (v2 §8, 방어 7번).

흔한 함정
  • anti-windup 없이 포화를 겪으면 회복이 이상하게 느리다 — 버그가 아니라 windup이다. 끄고 켜서 비교하라.
  • 시간 스텝을 크게 잡으면(예: 10분) 멀쩡한 게인도 발산한다 — 이산화의 한계. dt를 줄여 확인.
  • 열화(느린 추세)와 외란(부하 변동)을 혼동 — 경보 검증 시나리오에 부하 스텝 변화를 넣어 FP가 튀지 않는지 확인하라.
  • 정상 구간 통계에 warm-up을 포함시키면 관리한계가 넓어져 recall이 죽는다 (Step 2의 교훈 재등판).
  • 범위 팽창 — 전 스테이션 열 모델, 2차 모델, 자동 튜닝은 전부 확장이다. v2 원칙: ST2 1개, 1차 모델 고정.
MISSION 02b

반나절 실습 — v2 §4.2(e)의 초안 만들기

  1. 플랜트만 개루프(u 고정)로 돌려 부하 변화에 온도가 표류하는 것을 확인한다.
  2. PID를 닫고 수동 튜닝 절차(P→I→D)를 밟는다. 각 단계의 정착시간·오버슈트·정상상태 오차를 표로 기록.
  3. anti-windup을 껐다 켜서 포화 후 회복 차이를 그래프로 남긴다.
  4. 열화 시나리오를 돌려 온도·냉각 출력·k_cool 3단 그래프로 "방어기 → 포화 → 이탈"을 재현한다.
  5. SPC ±3σ 경보(온도·제어 노력 각각)를 구현하고 precision/recall/오경보/리드타임 표를 만든다 — 제어 노력 경보가 얼마나 먼저 울리는지가 핵심 결과.
  6. 이 코드를 ThermalPID 클래스로 정리해 Step 1의 SimPy 미니 라인 ST2에 붙이고, 온도·출력 이벤트를 Step 3의 InfluxDB 스키마에 추가한다.
