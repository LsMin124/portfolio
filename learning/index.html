IM-Twin 사전 학습 v2 기획서 기반 · 10주 프로젝트 대비

Intelligent Manufacturing Twin — Prerequisites

프로젝트를 시작할 수 있는 몸을 만드는
9단계 사전 학습 로드맵

각 스텝의 실습 미션은 그 자체가 프로젝트 부품의 프로토타입이 되도록 설계했다. 학습이 끝나면 버릴 코드가 아니라, 1주차부터 바로 이어 쓸 수 있는 초석이 남는다. 순서는 v2 기획서의 10주 로드맵과 정렬되어 있다.

전체 진행률 0%
PHASE A

워밍업 — 시뮬레이션의 뼈대

프로젝트 착수 전

  • 시스템 전체 그림과 용어 지도

    v2 아키텍처를 한 장으로 해부한다. 반송 오더 하나의 일생을 따라가며 모든 구성요소의 역할을 파악.

    개념 ~2h 0/5

  • SimPy 이산사건 시뮬레이션

    env·프로세스·리소스·스토어로 미니 생산 라인을 세운다. 미션 결과물이 곧 DES 코어의 원형.

    코딩 ~10h 0/6

  • 생산시스템 이론 최소 필수

    Little의 법칙, M/M/1, 병목, 반복 실행과 신뢰구간. what-if 응답의 수학적 근거를 만드는 스텝.

    이론+검증 ~6h 0/6
PHASE B

파이프라인 — 데이터가 흐르는 길

프로젝트 1–2주차 병행

  • InfluxDB + Grafana + Docker Compose

    시계열 데이터 모델(measurement·tag·field)과 대시보드. SimPy 이벤트를 적재해 처리량 패널까지.

    인프라 ~5h 0/6
PHASE C

로봇·3D — 트윈의 몸

ROS2는 1–2주차 병행, Isaac은 3주차 직전

  • ROS2 기초 — 노드·토픽·rclpy

    pub/sub 통신 모델과 CLI 도구. 미션의 오더 발행/수행 노드 한 쌍이 DES↔Isaac 브리지의 원형.

    코딩 ~12h 0/6

  • Isaac Sim 입문 — USD·자산·ROS2 브리지

    Omniverse 개념, 창고 씬과 AMR 자산, ROS2 토픽 연결까지. 강등 게이트 대비 Gazebo 경로도 정리.

    3D+통합 ~14h 0/6
PHASE D

인터페이스·에이전트 — 트윈의 입과 뇌

프로젝트 6주차 전후

  • OPC UA 개념 + asyncua 경량 서버

    주소공간·노드 모델만 정확히 잡고, 트윈 상태를 노출하는 미니 서버를 만든다. 경량 원칙 유지.

    인터페이스 ~4h 0/5

  • LangGraph 에이전트 — 그래프·도구·승인 게이트

    상태 그래프, 도구 호출, 인터럽트. 가짜 what-if 도구로 라우팅하는 미니 에이전트를 만든다.

    코딩 ~10h 0/6
PHASE E

품질 — 증명 가능한 에이전트

프로젝트 8주차 전후

  • LLM 에이전트 평가 방법론

    평가셋 설계, 라우팅 정확도·groundedness·게이트 준수율 정의와 채점 하네스 구조.

    방법론 ~5h 0/5
사용법

RULES OF ENGAGEMENT
  • 순서 고정, 깊이 가변 — 스텝 순서는 지키되, 이미 아는 내용은 체크리스트로 자가 점검만 하고 넘어간다.
  • 실습 미션이 본체 — 읽기만 하고 넘어가면 남는 게 없다. 미션 산출물은 ~/imtwin-warmup/ 같은 연습 저장소에 커밋해둔다.
  • 막히면 시간 박스 — 한 이슈에 2시간 이상 막히면 기록하고 다음으로. 완벽주의는 v2 기획서 §9의 최대 리스크다.
  • 체크 상태는 이 브라우저의 localStorage에 저장된다 (다른 브라우저·시크릿 모드에서는 초기화).
