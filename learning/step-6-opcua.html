OPC UA — 트윈에 산업 표준 창구 달기 (경량)

인터페이스 ~4h v2 §4.2(c) — 경량 원칙

학습 목표

  • OPC UA가 무엇이고 이 프로젝트에서 왜 "경량"으로만 쓰는지 설명할 수 있다.
  • 주소공간·노드·NodeId 개념으로 서버 구조를 읽을 수 있다.

프로젝트와의 연결 — v2 §4.2(c), 면접 방어 4번

"PLC 없는 트윈이 현실성 있나?"에 대한 답이 이 창구다. 트윈 상태를 OPC UA로 노출하면 "실물 설비·상위 시스템과 연동 가능한 표준 인터페이스를 갖췄다"고 말할 수 있다. 단, v2의 결정은 경량 유지 — 정보 모델을 파고들지 않고 창구의 존재만 증명한다. 여기 시간을 쓰는 만큼 지능형제조 코어(what-if·에이전트)에서 시간이 빠진다는 걸 기억하라.

1. OPC UA 3분 요약

OPC UA는 산업 장비의 데이터와 제어를 표준화된 트리(주소공간)로 노출하는 프로토콜이다. 장비(또는 트윈) 쪽이 서버가 되어 자기 상태를 노드로 공개하고, SCADA/MES/분석 시스템이 클라이언트로 접속해 브라우즈·읽기·쓰기·구독한다. 벤더 중립이라 "어느 회사 장비든 같은 방식으로 읽는다"는 게 존재 이유다.

우리 트윈이 서버가 된다 = 가상 공장이 실제 공장 설비와 똑같은 방식으로 말을 건다는 뜻. 이식 가능성 서사가 여기서 나온다.

2. 주소공간 — 폴더 트리라고 생각하면 80%는 맞다

  • 서버의 루트 아래 Objects 폴더가 있고, 그 밑에 Object(폴더 역할)와 Variable(값)이 트리를 이룬다.
  • 모든 노드는 NodeId(네임스페이스 인덱스 + 식별자)로 유일하게 지목된다 — 예: ns=2;i=15.
  • 클라이언트는 트리를 브라우즈해서 구조를 발견하고, 관심 변수를 구독하면 변경 시 알림을 받는다.

우리가 노출할 최소 트리:

Objects └── Line1 ├── ThroughputPerHour (Double, 읽기) ├── WIP (Int64, 읽기) ├── ActiveOrders (Int64, 읽기) └── TargetAmrCount (Int64, 쓰기 가능 — 승인 게이트 제어 지점)

3. asyncua — 미니 서버 만들기

asyncua는 asyncio 기반이다. async/await가 처음이면 "await = 이 작업이 끝날 때까지 다른 일 하러 감"으로 읽고 시작하면 된다:

twin_opcua_server.py
import asyncio
from asyncua import Server

async def main():
    server = Server()
    await server.init()
    server.set_endpoint("opc.tcp://0.0.0.0:4840/imtwin/")

    ns = await server.register_namespace("http://imtwin.example/factory")

    line = await server.nodes.objects.add_object(ns, "Line1")
    throughput = await line.add_variable(ns, "ThroughputPerHour", 0.0)
    wip        = await line.add_variable(ns, "WIP", 0)
    target_amr = await line.add_variable(ns, "TargetAmrCount", 2)
    await target_amr.set_writable()      # 이 노드만 클라이언트 쓰기 허용

    async with server:
        tick = 0
        while True:
            await asyncio.sleep(5)
            tick += 1
            # 학습용 합성값 — 프로젝트에선 DES 통계가 들어온다
            await throughput.write_value(10.0 + (tick % 5) * 0.5)
            await wip.write_value(3 + tick % 4)

asyncio.run(main())

프로젝트에서는 합성값 자리에 DES 코어의 실시간 통계가 들어가고, TargetAmrCount 같은 쓰기 가능 노드가 에이전트 승인 게이트 제어의 물리적 종착지가 된다.

4. 확인 — 클라이언트로 접속해보기

  • UaExpert(무료 GUI 클라이언트)를 설치해 opc.tcp://localhost:4840/imtwin/에 접속 → Objects 트리 브라우즈 → 변수를 Data Access View에 끌어다 놓으면 5초마다 갱신되는 게 보인다.
  • 파이썬으로도 확인 가능: asyncua.Client로 접속해 get_node → read_value() 몇 줄이면 된다 — 에이전트의 제어 도구가 이 클라이언트 코드의 연장이다.
흔한 함정 + 경량의 경계선
  • 학습 서버는 익명 접속(보안 없음)이다 — 로컬 학습용으로만. README에 "보안 모드는 범위 외, 실배포 시 인증서 기반 필수"라고 명시하는 것 자체가 이해의 증거다.
  • 갱신 루프에서 time.sleep을 쓰면 서버 전체가 멈춘다 — 반드시 await asyncio.sleep.
  • 포트 4840이 이미 점유되어 있으면 조용히 실패할 수 있다.
  • 여기까지가 경량이다 — 컴패니언 스펙, ISA-95 전체 계층, 이력(Historical Access)은 전부 확장 목록. 파고들기 시작하면 v1의 무게중심으로 회귀하는 것이다.
